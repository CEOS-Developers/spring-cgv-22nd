# 외부 API를 호출하는 방법

- **HttpURLConnection/URLConnection
  :** HTTP 관련 기능을 지원하는 URLConnection으로 자바에서 제공하는 기본적인 API이다.

  순수 자바로 HTTP 통신이 가능하며, url을 통해 외부 API에 연결을 하고 데이터를 전송할 수 있다.

  장점

    - 객체 생성이 쉽다
    - 프레임워크에 상관없이 사용이 가능하다.

  단점

    - 오래된 자바 버전에서 사용하는 클래스로 동기적 통신을 기본으로 사용한다.
    - 동기적이므로 요청을 보내고, 응답을 받을 때까지 스레드가 대기 상태에 있다.
    - 기본적인 요청/응답기능이 제공되지만, 추가적인 기능들에 대해선 직접 구현해야 된다.

- **RestTemplate**

  : 스프링에서 제공하는 HTTP 요청 처리를 위한 클라이언트 라이브러리

  다양한 HTTP 메서드를 활용하며 원격 서버와 동기식 방식으로 통신한다.

  ✅ 동기식 방식으로 요청을 보내고 응답을 받을 때까지 블로킹되며, 요청과 응답이 완료되기 전까지 다음 코드로 진행되지 않음

  동기식요청? 비동기식 요청?

  | **비교** | **동기식(Synchronous)** | **비동기식(Asynchronous)** |
      | --- | --- | --- |
  | 실행 모델 | 요청과 응답이 동시에 일어남 | 요청과 응답이 동시에 일어나지 않음 |
  | 블로킹 | 요청에 대한 응답이 올 때까지 블로킹 됨 | 요청에 대한 응답을 기다리지 않고 다른 작업을 수행할 수 있음 |
  | 성능 | 대량의 요청이 동시에 발생하면 성능 저하가 발생할 수 있음 | 대량의 요청을 효율적으로 처리할 수 있음 |
  | 코드 복잡도 | 비교적 간단하고 직관적임 | 코드가 복잡해지고 디버깅이 어려움 |

  장점

    - 단순한 구조로 빠른 구현이 가능하다

  단점

    - 블락킹 I/O를 사용하기에, 트래픽이 많거나 동시성 문제가 많은 프로젝트에선 성능이 좋지않다.

  ❗️Spring 5.0부터 WenClient가 나오면서, 현재는 WebClient가 더 많이 쓰이는 추세(Spring doc에서도 WebClient를 추천함)

- **HttpClient**

  : 자바 애플리케이션이 다른 서버나 API에 HTTP 요청을 보내고 응답을 받기 위한 라이브러리 또는 인터페이스

  동기와 비동기 방식 모두 지원하며, HTTP/2와 WebSocket 등을 사용할 수 있다.

- **WebClient**

  : Spring 5부터 등장한 비동기/논블로킹방식도 지원하는 HTTP 클라이언트

  장점

    - 리액티브 프로그래밍과 연계하여, 대량 동시 요청이나 스트리밍 처리에 유리
    - 함수형 API로 코드가 깔끔하고 오류를 처리하기 편하다

  단점

    - 리액티브 프로그래밍으로 인해, 학습곡선이 가파르다 (학습 초기에 어려움을 많이 느끼고, 숙달되기까지 상당한 시간과 노력이 필요하다)
    - 사용하려면 webflux 라이브러리를 사용해야해서, 소규모에서는 좋지 않을 수 도 있다.
- **OpenFeign**

  : 선언적 접근 방식을 활용한 클라이언트
  장점

    - 인터페이스와 어노테이션만으로 HTTP API 호출을 간단하게 작성할 수 있다.
    - 로드밸런싱 기능을 자체적으로 제공 (대규모서비스에 👍)

  단점

    - RestTemplate과 동일하게 Blocking I/O를 사용하여 트래픽이 많거나 동시성 문제가 많은 프로젝트에선 성능이 좋지 않다.

⇒ 따라서 WebClient 사용
이유: restTemplate, OpenFeign은 동시성문제가 많은 프로젝트에서 성능이 좋지않다. cgv의 경우 영화 예매 시 동시성 문제가 많을 거로 예상되어 선택X, 그리고 webClient가 webFlux때문에 소규모 프로젝트에선 좋지 않을 수 도 있지만 cgv는 대규모 프로젝트라고 생각되어, 충분히 사용해도 된다고 판단하였다
